# المسرد: الكلمات الصعبة

جدول سريع لكل المصطلحات التي قد تسمعها في نسّاج. كل واحد بجملة واحدة واضحة.

| المصطلح | الشرح |
|---|---|
| **المنسّق** | الذكاء الذي يستقبل طلبك ويوزّعه على الوكلاء — مثل مدير المشروع. |
| **الوكيل** | خبير واحد متخصص: مبرمج، مصمم، محلل. يفعل الشغل الفعلي. |
| **جلسة** | محادثة مستمرة واحدة حول مهمة — قد تستمر دقائق أو ساعات. |
| **مشروع** | مجلد عمل كامل: ملفات، كود، تاريخ، ومهام. مثل nassaj-dev. |
| **لوحة المشروع** | جدول حي يلخص حالة المشروع: المراحل، المهام، الحالات. |
| **المرحلة** | قسم كبير من المشروع (مثل: التصميم، الترميز، الإطلاق). |
| **السبرنت** | دفعة عمل قصيرة (أسبوع تقريباً) لإنجاز مجموعة مهام محددة. |
| **المهمة** | واحدة من المهام الصغيرة (مثل: "برمج زر الدفع"). |
| **ADR** | قرار معماري موثّق عن بنية المشروع (مثل: اختيار React). |
| **حصة الاستخدام** | ميزانية الساعات: كل موظف عنده ٥ ساعات بالجلسة + ساعات أسبوعية. |
| **نافذة السياق** | مساحة الذاكرة التي يحتفظ بها نسّاج من الجلسة (إذا امتلأت توقّف). |
| **commit** | تسجيل تغيير في الكود مع رسالة توثّق ماذا تغيّر. |
| **build** | عملية بناء الكود وتحويله لموقع مشغّل. |
| **restart** | إعادة تشغيل الخادم أو التطبيق. يوقّفه ثم يشغّله من جديد. |
| **deploy** | نشر التطبيق على الخادم الفعلي (الإنتاج). |
| **backend** | الجزء الخفي: الخوادم والقواعد البيانات وحساب العمليات. |
| **frontend** | الجزء الظاهر: الصفحات والأزرار والألوان التي ترى. |
| **API** | جسر يتحدث من خلاله frontend و backend مع بعضهم. |
| **قاعدة البيانات** | خزانة ضخمة تحفظ كل البيانات (مستخدمين، مبيعات، إلخ). |
| **RTL** | اليمين لليسار — طريقة عرض الواجهة بالعربية (اليمين أولاً). |
| **UI** | واجهة المستخدم — كل ما ترى: أزرار، نصوص، صور. |
| **UX** | تجربة المستخدم — كيف تشعر عند استخدام الموقع. |
| **تصميم** | رسم الموقع قبل البرمجة: الألوان، التخطيط، الشكل. |
| **برمجة** | كتابة الكود الفعلي بلغات مثل JavaScript أو Python. |
| **اختبار** | فحص الموقع للتأكد من أنه يشتغل بلا أخطاء. |
| **خطأ/Bug** | شيء ما ما يشتغل كما هو مفروض. |
| **إصلاح** | تصحيح الخطأ. |
| **الكِندِي** | الشركة الأم والعلامة الوحيدة والهوية الموحّدة والمرجعية للقرارات الكبيرة. |
| **دار الكِندِي** | الذراع الوحيد من الكِندِي: الخدمات الاستشارية والتطوير المخصص للعملاء. |
| **الكِندِي مفتوح المصدر** | وسم وظيفي (ليس قسماً): تصنيف لمخرجات دار الكِندِي التي تُنشر مفتوحة المصدر (github.com/AlKindy-OSS). |
| **نسّاج** | أداة ذكاء اصطناعي داخلية لا تُباع ولا تظهر للعملاء — تخدم الفريق الداخلي فقط. |
| **نسّاج كور** | العقل: المنسّق والوكلاء والقواعد التنظيمية والقوانين. |
| **نسّاج ديف** | الواجهة: الموقع الذي تفتحه بالمتصفح على nassaj-dev.alkindy.tech. |
| **هوية بصرية** | الألوان والخطوط والشعار الموحّد للشركة. |
| **branded house** | نمط تصميم واحد موحّد لكل المنتجات والخدمات (طريقة الكِندِي). |
| **token** | وحدة تقاس بها تكاليف استخدام الوكلاء (مثل كلمات مكتوبة). |
| **cache** | ذاكرة مؤقتة سريعة تحفظ نسخة من البيانات الكثيرة الاستخدام. |
| **session** | جلسة عمل — نفس معنى جلسة أعلاه. |
| **user/موظف** | أي شخص يستخدم نسّاج (أنت ومزملاؤك). |
| **أفاتار** | صورة صغيرة تمثلك بجانب اسمك. |
| **شارة** | علامة صغيرة بجانب الاسم توضح الدور (مثل: مالك، محرر). |
| **أرشفة** | إخفاء شيء عن القائمة الرئيسية، لكن يبقى موجود للبحث. |
| **تحديث حي** | الموقع يتحدّث تلقائياً بلا تحديث يدوي. |
| **WebSocket** | تقنية تسمح للموقع بإرسال بيانات جديدة فوراً. |
| **قراءة فقط** | إذن محدود: تستطيع تقرأ لكن ما تستطيع تعدّل. |
| **صلاحية** | الإذن الذي يحدد ماذا تستطيع تعمل في الموقع. |
| **تشفير** | تحويل البيانات لصيغة غير مفهومة حماية من الهاكرز. |
| **كلمة المرور** | رمز سري يثبت هويتك (تشفّر قبل ما تُحفظ). |
| **ثيم** | نمط ألوان كامل (فاتح، داكن، الكِندِي). |
| **placeholder** | صورة أو مكان فارغ تحت الإنشاء (قيد التحضير). |
| **قالب** | نموذج جاهز تستطيع تنسخه وتعدّله. |
| **توثيق** | كتابة شرح عن الكود أو الميزة. |
| **commit message** | الرسالة التي تكتبها مع التغيير (توضّح ماذا غيّرت). |
| **git** | نظام تتبع الكود — يحفظ كل تغيير وتاريخه. |
| **GitHub** | الموقع الذي يحفظ فيه الكود (مثل Google Drive بس للبرمجة). |
| **اختبار آلي** | برنامج يختبر الكود تلقائياً بلا تدخل يدوي. |
| **جودة** | مستوى نظافة الكود والأداء والأمان. |
| **performance** | سرعة الموقع — كم بسرعة يحمل وينفّذ. |
| **أمان** | حماية البيانات من المخترقين والأخطاء. |
| **CRUD** | أربع عمليات أساسية: Create (انشئ)، Read (اقرأ)، Update (عدّل)، Delete (احذف). |
| **endpoint** | عنوان محدد في الخادم يستقبل طلبات. |
| **request/response** | الطلب الذي يرسله الموقع والرد الذي يأتي من الخادم. |
| **استدعاء** | طلب رسمي من المنسّق للوكيل: "افعل هذا من فضلك". |
| **تفويض** | إعطاء شخص إذن بفعل شيء (مثل: فوّضت الوكيل بالبرمجة). |
| **موافقة** | قبول رسمي من المالك أو صاحب السلطة. |
| **قرار** | اختيار رسمي بين بدائل متعددة (مثل: قررنا استخدام React). |
| **سياسة** | قاعدة موثّقة يجب اتباعها (مثل: كل كود جديد يحتاج اختبار). |
| **استراتيجية** | خطة طويلة الأمد توضح الاتجاه (مثل: نركز على السوق الخليجية). |
| **ميزة** | شيء جديد نضيفه للموقع. |
| **تحسين** | جعل شيء موجود أفضل وأسرع. |
| **تصحيح** | إصلاح خطأ أو مشكلة. |
