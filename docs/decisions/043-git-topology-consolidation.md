## Context

- **الريبو المحلي `shallow` (و`detached HEAD`):** `git rev-parse --is-shallow-repository` = `true`. التاريخ المبتور جعل أدوات القياس تُظهر فجوة مضلِّلة بين `main` وخط التطوير، فبدا أن `main` «أحدث» بينما الواقع عكسه.
- **`main` متقدّم بـ commitين merge فقط** عن نقطة التفرّع (`merge-base` = `ae1d7ca`): `873ea00d` (#1) و`2d7163c5` (#2)، وكلاهما دمج لفرع `chore/sync-upstream-v1.33`. التحقّق: `git diff ae1d7ca 2d7163c5` **فارغ** — أي **صفر تغيّر ملفات على جانب `main`** منذ نقطة التفرّع (الـ merge-commits لا تُدخل صافي تغيير مقابل merge-base).
- **`origin/verify/b63-live-counter-pkgC` (رأسه `42cc5ee`) = خط تطوير نسّاج الكامل المدفوع:** ‏**360 commit** متقدّم عن `main`. يضمّ B-80a/b/c (replay + ghost-detach، ADR-041/042) + B-63/74/75/70 + دمج upstream-sync. **الأجهزة (dev/الرخيمي) تشغّل هذا الخط فعلياً** عبر `dist`/`dist-server` المبنيّين منه.
- **إصلاح Fable (`df53195f`, 5 ملفات نظيفة):** يخفي `claude-fable-5` غير المُطلَق من قائمة النماذج (ADR-044). مبنيّ محلياً فوق commits جلسة موازية (T-202/T-203 غير مدفوعة)، لكن الملفات الخمسة لم تُلمس في T-202/203 ⇒ يُطبَّق cherry-pick نظيفاً فوق `42cc5ee`.
- **جلسة Claude موازية نشطة** على working tree الرئيسي وقت التنفيذ — فُرض العمل كاملاً في worktree معزول تحت `/tmp`، وبقي عملها (T-202/203) لها دون مساس.

## Alternatives

1. **rebase خط b63 فوق `main`** — مرفوض: يعيد كتابة 360 SHA على فرع مدفوع تشغّله الأجهزة، يكسر مراجع `dist`/transcripts/الجلسات، ويخاطر بفقد تاريخ replay/detach الدقيق.
2. **squash-merge** — مرفوض: يطمس 360 commit (ومنها ADR-041/042 وسلسلة B-80) في commit واحد، فيُفقد التاريخ التشخيصي الذي تعتمده الحوادث المستقبلية.
3. **fast-forward لـ `main` إلى `42cc5ee`** — غير ممكن نظيفاً: `main` فيه commitا merge ليسا على خط b63، فلا ff مباشر.
4. **اعتماد b63 فرعاً رئيسياً جديداً وهجر `main`** — مرفوض: يكسر `origin/HEAD` وروابط PR/CI القائمة على `main`، ويربك المتعاونين.
5. **المعتمد (أدناه): merge-commit من خط b63 إلى `main`** — يحفظ كل تاريخ الطرفين بلا إعادة كتابة، وبما أن جانب `main` صفر-تغيير منذ merge-base فالدمج بلا تعارض.

## Decision

1. **دمج خط تطوير نسّاج إلى `main` بـ merge-commit** (لا rebase، لا squash، لا fast-forward قسري).
2. **إصلاح Fable يُطبَّق بـ cherry-pick على نفس الخط** (`df53195f` → `e01ed472` فوق `42cc5ee`) قبل الدمج، فلا يعتمد على commits T-202/203 غير المدفوعة.
3. **عمل الجلسة الموازية (T-202/T-203) يبقى لها** — لا يُلتقَط ولا يُدمَج هنا؛ هي تدمج تعديلاتها بنفسها.
4. **سياسة Git مُثبَّتة بعد هذا القرار:**
   - فرع عمل **مسمّى** دائماً، لا العمل على `detached HEAD`.
   - **تنظيف worktrees** بعد كل workflow (إزالة worktrees المؤقتة فور انتهاء المهمة).
   - أي workflow على الريبو الرئيسي حين وجود جلسة موازية ⇒ worktree معزول تحت `/tmp` حصراً، دون لمس working tree الرئيسي.

## Consequences

- **`main` = خط الحقيقة الموحَّد** بعد الدمج: يحوي B-80 (ADR-041/042) + إخفاء Fable (ADR-044) + كامل خط التطوير.
- **لا فقد تاريخ:** كل الـ 360 commit + commitا merge على `main` محفوظان؛ التشخيص المستقبلي للحوادث يبقى ممكناً.
- **التحقّق قبل الدمج (في worktree `/tmp`):** `build:server` و`build:client` و`npm test` اجتازت كلها (تفاصيل في CHANGELOG/PR) — لا انحدار.
- **الريبو shallow يبقى شركاً تشخيصياً:** أي قياس فجوة مستقبلي يجب أن يحسب أن المحلي مبتور التاريخ، أو يُجرى `git fetch --unshallow` قبل الاستنتاج.
- **متبقٍّ للمالك (خارج نطاق هذا الـADR):** نشر `main` المُوحَّد على الأجهزة (build + pm2 restart) — لم يُنفَّذ هنا.
