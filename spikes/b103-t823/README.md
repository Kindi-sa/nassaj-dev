# B-103 / T-823 — التصليب + رؤية safe-restart/الدرين + soak الظل + بوابة التفعيل

المرحلة 5 (الأخيرة) من B-103. الأعلام `WORKFLOW_SUPERVISOR` و`WORKFLOW_SUPERVISOR_CHAT_LOCK`
تبقى **OFF**؛ كل ما هنا يُثبَت على **نسخة ظل** معزولة (لا يمسّ العملية/القاعدة/dist-server
الحيّة إلا قراءةً لاعتماد المالك عبر symlink، كما في harness T-820/821/822).

> **هذا soak ظل — بديلٌ مُثبِت للـ72h الحي، وليس الـsoak الإنتاجي.** الـ72h الحي +
> توقيع devops/qa-critic في **بوابة التفعيل** (`docs/plans/B-103-ACTIVATION-GATE-2026-07-10.md`)،
> وهو بوابة مالك لا يمكن إتمامها في جلسة المنسّق (safe-restart يؤجّل بوجود جلسة حيّة، وrestart محجوب).

## المحتويات

| المسار | الغرض |
|---|---|
| `soak/soak.sh` | soak الظل الموسّع: ≥30 مهمة عبر الأصناف الثلاثة، دورات إطلاق→اكتمال→تسليم مع restart متكرر وتزامن. يقيس: تسريب عملية/وحدة، تسليم مزدوج/ضائع، ثبات الذاكرة، سلامة القفل. |
| `criteria/reboot-orphan.sh` | إثبات دلالات reboot: الوحدة العابرة لا تنجو reboot ولا تُستأنف تلقائياً ⇒ «يتيمة-مرئية» (بطاقة CRASHED + سجل `reboot-orphan` resumed:false)، بلا إعادة تشغيل. |
| `criteria/safe-restart-visibility.sh` | إثبات رؤية safe-restart لوحدات `wf-*.service` (قراءة-فقط): تظهر عند العلم ON، تصمت تماماً عند OFF (byte-identical)، ولا تلمس الوحدة. |
| `evidence/soak-run.log` | مخرَج تشغيل soak محفوظ (الأرقام). |

## التشغيل

```bash
# soak الظل الكامل (يُنشئ نسخة ظل على منفذ/قاعدة/state مؤقت، ويهدمها بـtrap مضمون)
bash spikes/b103-t823/soak/soak.sh

# إثبات دلالات reboot (bring-up خفيف: قاعدة مؤقتة + مشرف، بلا سيرفر/claude حقيقي)
bash spikes/b103-t823/criteria/reboot-orphan.sh

# إثبات رؤية safe-restart (مستقل: يُنشئ وحدة wf-*.service حقيقية بـsystemd-run sleep)
bash spikes/b103-t823/criteria/safe-restart-visibility.sh
```

كل السكربتات ذاتية-الهدم (trap على EXIT/INT/TERM): توقف المشرفين، تُنظّف كل `wf-*.service`،
وتحذف الشجرة المؤقتة. الاعتماد على harness T-821 (`spikes/b103-t821/`) لبناء السيرفر+المشرف الحقيقيين.

## ما يثبته soak (§و/م5 المتاح دون تفعيل حي)

- **صفر تسريب عملية/وحدة:** عدّ `wf-*.service` (كل الحالات) + عمليات المشرف + flock — الأساس == بعد.
- **صفر مزدوج/صفر ضائع:** الـjsonl مصدر الحقيقة (عدّ البطاقات لكل محادثة == المهام الفريدة)، عبر ≥30 مهمة
  وrestart متكرر (kill -9 وسط التسليم عند إزاحات متفاوتة).
- **ثبات الذاكرة:** RSS لمشرف واحد طويل العمر عبر دورات متعددة (لا نمو متراكم).
- **سلامة القفل:** مالك-واحد (المثيل الثاني يخرج بهدوء)؛ بعد كل kill -9 يعيد المثيل الجديد اكتساب القفل (لا قفل بائت).

نتائج الأصناف الثلاثة (SUCCEEDED/PARTIAL/CRASHED) مشتقّة من **مخرَج claude حقيقي** (fixtures T-819)
لا مصطنع (درس 6.5%، `feedback_synthetic_fixtures_false_confidence`). عددٌ **معتدل** من الإطلاقات
الحقيقية (Phase C) يمارس المنتِج ودورة حياة الوحدة حيّاً.
